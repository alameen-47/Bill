# Bug Fixes - COMPLETED ✅

## Issues Fixed:

### 1. Login.jsx - Duplicate className and style props
- **Problem**: Components had both `className` (Tailwind) and `style` props simultaneously, which causes React Native errors
- **Fix**: Removed all `className` props and consolidated styling into `style` props only

### 2. NavigationBar.jsx - Duplicate className and style props
- **Problem**: Components had both `className` and `style` props
- **Fix**: Removed all `className` props and used only `style` props for React Native compatibility

### 3. Settings.jsx - iOS PermissionsAndroid crash
- **Problem**: Used `PermissionsAndroid.request()` on iOS which is Android-only and causes crashes
- **Fix**: Added Platform check - only request Android permissions, iOS permissions handled by react-native-bluetooth-classic

### 4. Info.plist - Missing iOS permissions
- **Problem**: Missing required iOS permissions for camera, photo library, Bluetooth, and location
- **Fix**: Added the following iOS permissions:
  - NSCameraUsageDescription
  - NSPhotoLibraryUsageDescription
  - NSPhotoLibraryAddUsageDescription
  - NSBluetoothAlwaysUsageDescription
  - NSBluetoothPeripheralUsageDescription
  - NSLocationWhenInUseUsageDescription
  - UIBackgroundModes (bluetooth-central)

### 5. AppStack.jsx - Initial route set to Reciept
- **Problem**: Initial route was set to "Reciept" instead of "HomeScreen"
- **Fix**: Changed initialRouteName from "Reciept" to proper auth flow (handled by AppNavigator)

## Summary of Changes:

1. **src/screens/Login.jsx** - Fixed duplicate className/style props
2. **src/components/NavigationBar.jsx** - Fixed duplicate className/style props  
3. **src/screens/Settings.jsx** - Fixed iOS PermissionsAndroid crash
4. **ios/Bill/Info.plist** - Added iOS permissions for camera, Bluetooth, location
5. **src/navigation/AppStack.jsx** - Fixed initial route

## Notes:
- NativeWind/Tailwind className props don't work in React Native - use style props only
- Always check Platform.OS before using Android-specific APIs
- iOS requires explicit permission descriptions in Info.plist
